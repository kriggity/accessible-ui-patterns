Accessible UI Patterns

A small set of vanilla JavaScript interaction patterns built with accessibility and UX correctness as first-class concerns.

This project focuses on common UI patterns that are frequently implemented incorrectly, and demonstrates how they can be built with clear semantics, predictable behavior, and documented tradeoffs.

Patterns

  • Button

    Baseline semantic button behavior using native HTML.

  • Modal / Dialog

    Accessible dialog pattern with focus management and keyboard support.

  • Tabs

    ARIA-compliant tabbed interface following authoring practices.

  • Form Error Messaging

    Accessible validation and error messaging patterns.

Scope & Constraints

  • No frameworks or build tooling
  • Minimal CSS and no theming
  • Patterns are intentionally limited in scope
  • Documentation is part of the deliverable

Accessibility

Each pattern is tested with keyboard navigation and common screen readers. Accessibility behavior and known limitations are documented per component.

Using this repository

Open this file in a browser and navigate to individual patterns. Each pattern includes example markup, JavaScript, accessibility notes, and documented tradeoffs.

Built as a focused exploration of accessible interaction patterns, not a production-ready component library.
